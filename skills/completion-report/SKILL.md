# Completion Report Format

작업 완료 시 아래 형식을 따른다.

```text
완료했습니다.

변경 사항
- <파일>: <내용>

검증
- <명령 또는 확인>: 성공/실패/미실행 사유

주의/다음 단계
- <필요 시>

사용량
- Claude Code: <수치 또는 확인불가>
- Codex/OMX: <수치 또는 확인불가>
```

## 규칙

- 검증 명령이 없거나 실패하면 `미실행/실패 사유`를 명시한다.
- 사용량 확인이 불가능하면 수치를 추측하지 말고 `확인불가`로 표기한다.
- 완료 보고에는 변경 파일, 검증 결과, 남은 리스크를 포함한다.
